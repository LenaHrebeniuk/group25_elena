Крaткое описание (Summary)  - поле, в котором нужно поместить весь смысл баг репорта.

Проект (Project) - название текущего проекта

Компонент приложения (Component) - в котором возник дефект

Номер версии (Version) - версия на которой была найдена ошибка 

Серьезность (Severity) - система градации серьезности дефекта:
S1 Блокирующий (Blocker)
S2 Критический (Critical)
S3 Значительный (Major)
S4 Незначительный (Minor)
S5 Тривиальный (Trivial)

Приоритет (Priority) - приоритет дефекта:
P1 Высокий (High)
P2 Средний (Medium)
P3 Низкий (Low)

Статус (Status) - отображает статус бага в своем жизненном цикле

Автор (Author) – автор баг репорта

Назначен на (Assigned To)  - кто должен исправить дефект

Окружение – в каких условиях был найден баг: версия ОС, браузера

Шаги воспроизведения (Steps to Reproduce) - писание пути, который приводит к возникновению дефекта

Фактический Результат (Result) - результат, к которому приходим выполнив все шаги воспроизведения

Ожидаемый результат (Expected Result) - результат, который будет в соответствии с требованиями

Прикрепленный файл (Attachment) - файл с логами, скриншот или любой другой документ, который может помочь прояснить причину ошибки или указать на способ решения проблемы
